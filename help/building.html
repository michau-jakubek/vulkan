Building the executable

After successful installation you can go to the building of the project. There is no single and unificated method how to do this. For an example if you prefer work on the Windows with Microsoft Visual Studio then it is enough that you will start the building process by clicking the build project from its menu. The second thing is that the Visual Studio has its own settings for the project type: Release and Debug, while under the Linux the project type is specified by some parameters passed to the command line, i.a. -ggdb. On the Linux system(s) basically the most convinient method is to go to the cmake's binary dir and type make command. Anyway, always you can also build the project using cmake only specifying the command:

  • cmake --build <${CMAKE_BINARY_DIR}> -j 8

This approach is good because in the case of Microsoft Visual Studio you don't have to launch it at all. You dont't have to specify any compilers manually, cmake will find a proper compiler and utilize it. In the cited build command I used also -j 8 parameter. It tells to the compiler how many threads it will use to build your project. In the case of building from scratch under the Linux with make you can also manipulate the number of running threads by specifying make -j 8.
