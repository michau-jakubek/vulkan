Launching

Eventually, after installation and building the best time has come up to launch an application. Typically, if you don't change anything, the build process created the app.exe on the Windows or app executable in the case of the Linux OS. Finally your application you can run by typing in the command line interpreter:

Linux:
Windows:
./app [app_params] <test_name> [test_params]
app [app_params] <test_name> [test_params]


Program options/parameters

The program options starts from -, sometimes from --, followed by the option name. One of them require a value and another don't. The value of the option is given after white space. If any option needs the value and it is not provided or a different option name is provided, then whatever follows the option name will be consumed and treated as the value, which will result in incorrect work of the program. Aany option can appear any number of times on the command line, with the last one taken into account.

The option list is usually followed by the name of the test and it separates the application options from the test options.

./app -d 1 -api 13 -spirv 16 -spvvalid triangle -t 2, where painted blue are application options, red is the test name and green are test parameters

.

Application ptions:

  -ll:                      prints available instance layer names

  -l-no-vuid-undefined:     suppress layers(s) VUID_Undefined warning

                            will match to VUID-vkBeginCommandBuffer-commandBuffer-00049
  -api <version>:           Vulkan API version to apply, (major * 10 + minor), default is 1.1

  -dprintf:                 enable Debug Printf feature

Compiler options:
  -vulkan <version>:        (major * 10 + minor), default is 11 aka vulkan1.0
  -spirv <version>:         (major * 10 + minor), default is 10 aka spirv1.0
                            Set the execution environment where the shaders will be executed in.
                            Defaults to :
                             * vulkan1.0 under --client vulkan<version>
                             * opengl    under --client opengl<version>
                             * spirv1.0  under --target - env vulkan1.0
                             * *spirv1.3  under --target - env vulkan1.1
                             * *spirv1.5  under --target - env vulkan1.2

  -spvdisassm:              generate SPIR-V dissassembly file
  -nowerror:                allows warnig(s) from external compilators

Most of the options are self-explanatory but several deserve the special attention. These are -d, -api, -vulkan, -spirv and -spvvalid.
